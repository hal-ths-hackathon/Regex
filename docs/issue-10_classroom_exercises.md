Issueタイトル: `[Backend/Frontend](10): 授業演習問題(Ex1〜Ex20)の統合と正規表現選択・入力モードの設計変更`

### 背景
現在のゲームシステムでは、選択肢モードにおいて「正解/ダミーの文字列そのもの」を選択肢として提示していました。これでは単なる文字列探しゲームになってしまうため、ゲーム本来の目的である「正規表現の学習」に沿って設計を変更します。
選択肢モードでも「正規表現パターン（例: `^[A-Z]{2}-[0-9]{2}[A-Z]-[0-9]{3}$`）の4択」を提示し、プレイヤーが正しい正規表現を選択し、それを適用した結果（`correct_string` のみに完全一致するか）で正誤判定を行う仕組みにします。
あわせて、授業で扱った20パターンの演習問題（Ex1 〜 Ex20）を問題テンプレートのバリエーションとして取り込みます。

### 実施内容

1. **授業の演習問題 (Ex1 〜 Ex20) のテンプレート実装 (`generator.py`)**
   - 20種類の問題テンプレート関数を定義し、それぞれ難易度（easy/hard）または問題内容に応じたデータを生成します。
   - 各関数が生成するデータ：
     - `hint`: 問題文（例: 「固定10桁。先頭２桁は任意英字（大文字限定）...」）
     - `correct_string`: 正解となる対象文字列（例: `IH-12A-000`）
     - `dummies`: 惜しいダミー文字列リスト（例: `ih-12a-000`, `IH12A000` など）
     - `correct_patterns`: 正解の正規表現パターンのリスト（例: `[r"^[A-Z]{2}-[0-9]{2}[A-Z]-[0-9]{3}$"]`）
     - `dummy_patterns`: 間違いの（惜しい）正規表現パターンのリスト（例: `[r"^[a-z]{2}-[0-9]{2}[A-Z]-[0-9]{3}$", r"^[A-Z]{2}-[0-9]{3}-[0-9]{3}$"]`）

2. **API レスポンスにおける `choices` フィールドの変更**
   - `choices` の中身を「文字列の選択肢」から **「正規表現パターンの選択肢」** に変更します。
   - `correct_patterns` からランダムに1つの正解パターンを、`dummy_patterns` からランダムに3つのダミーパターンを選び、計4つの選択肢としてシャッフルして返します。

3. **動作確認用デバッグ画面（`debug.html`）の更新**
   - UIの表示内容を「正規表現の4択」に更新します。
   - デバッグ画面上で、選択した（あるいは入力した）正規表現が `noise_text` に対してマッチングを試みるシミュレーション機能を追加し、最終的な正誤判定（正解文字列のみを抽出できるか）を視覚的にテストできるようにします。

4. **テストコード（`test_main.py`）の更新**
   - 新規追加した20パターンの全問題テンプレートが例外なく動的生成できることをテストします。
   - 選択肢（`choices`）が正規表現パターンであり、そのうち正解パターンが1つだけ含まれていることを検証するテストを追加します。

### 完了条件

- [x] デバッグ画面（`http://localhost:8090/debug`）で、正規表現パターンの4択から正解を選ぶ、またはテキスト入力することで、マッチのシミュレーションと正誤判定ができること。
