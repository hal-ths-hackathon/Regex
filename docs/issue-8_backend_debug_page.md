Issueタイトル: `[Backend/Frontend](8): バックエンド動作確認用デバッグ画面の実装（別ポート）`

### 背景
現在、ローカル開発環境においてポート競合等により、バックエンド（FastAPI）のAPI疎通や生成ロジックの動作確認がブラウザから行いにくい状態になっています。
バックエンドから生成される問題データ（正解文字列、ダミー、ノイズ）を個別に別ポート（例: ポート `8080`）で簡単にリクエストして確認できる、シンプルな動作確認用（デバッグ・管理用）のHTML画面および実行環境を準備します。

### 実施内容
- [ ] バックエンド（FastAPI）または新規の簡易Webコンテナで、ポート `8080` を用いてデバッグ用静的HTMLを配信する。
  - FastAPI自体に `/debug` や `/admin` のようなデバッグ用静的HTML配信エンドポイントを追加し、ホスト側ポート `8080` を割り当てる。
  - もしくは、docker-compose に 8080 ポートで動作する確認用コンテナを定義する。
- [ ] 動作確認用ページ（HTML/JS）の作成
  - 「問題生成 (Level: easy/hard)」ボタンを配置する。
  - ボタン押下時に `GET /api/v1/stages/generate?level=...` を呼び出し、レスポンス（`hint`, `correct_string`, `noise_text`）を整形表示する。
  - ノイズテキスト内の `correct_string` の位置などを視覚的に確認できるようにする。
- [ ] Docker / Compose 環境のポート構成の整理
  - 他のローカルプロセス（`palettewalk` 等）と衝突しないよう、ポート `8080` または `8009` など適切なポート割り当てを整備する。

### 完了条件
- [ ] ブラウザで `http://localhost:8080`（または指定のデバッグ用ポート）にアクセスした際、動作確認画面が表示されること。
- [ ] 画面上で難易度を指定して問題生成を実行でき、生成された JSON データが動的に表示されること。

### 影響範囲
- `docker-compose.yml` (ポート定義、環境変数など)
- バックエンドのルーティング、または新規動作確認用静的ファイルの追加
- API の CORS 設定
