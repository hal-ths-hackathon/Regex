Issueタイトル: `[Backend/Frontend](9): 回答選択肢および入力モードの生成ロジックの実装とデバッグ画面への統合`

### 背景
ゲームプレイ画面を実装するにあたり、単に問題（ノイズテキストと正解文字列）を生成するだけでなく、以下のゲーム仕様に対応するデータをAPIから提供する必要があります。
1. **選択肢モード**: 4つの選択肢（1つの正解選択肢と3つのダミー選択肢）を生成・提示する。
2. **入力モード**: ユーザーが入力した正規表現の妥当性チェックや、解答例となる正規表現パターンを提示する。

これらをバックエンド側で実装・拡張し、先ほど作成した動作確認用デバッグ画面（`debug.html`）をアップデートして、ブラウザから動作確認できるようにします。

### 実施内容
- [ ] バックエンド (FastAPI) で回答選択肢を生成するロジックの実装
  - 現在の `generate_stage` ロジックを拡張するか、新規APIエンドポイント（例: `/api/v1/stages/generate` のレスポンスを拡張）し、4択用の選択肢リスト（`choices` など）を含める。
  - 選択肢は「正解文字列」と「惜しいダミー」から動的に生成し、順序をシャッフルする。
- [ ] 入力モード向けの正解正規表現パターン（解答例）の提供
  - 問題の生成タイプ（郵便番号、携帯電話番号、日付、時刻）に応じた、正解となる代表的な正規表現パターン（例: `^\d{3}-\d{4}$` など）をレスポンスに含める。
- [ ] 動作確認用デバッグ画面（`debug.html`）のアップデート
  - 「回答の4択（どれが正解かも明示）」を表示するUIを追加する。
  - 「正解正規表現パターン」を表示するUIを追加する。
  - 実際にデバッグ画面上で動作確認が行えることを検証する。

### 完了条件
- [ ] `http://localhost:8090/debug` で問題を生成した際、4択の選択肢と正解インデックス、および正解正規表現パターンが画面上に動的に表示されること。
- [ ] バックエンドAPIのユニットテストに、新しく追加したフィールド（`choices` や `correct_patterns` など）のスキーマ検証・動作検証テストが追加され、パスすること。

### 影響範囲
