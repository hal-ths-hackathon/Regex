Issueタイトル: `[Frontend](7): スタートページ（トップ画面）の実装`

### 背景
ユーザーがアプリにアクセスした際に、最初に目にする「サービスの顔」となるスタートページを実装する。
独自の世界観（上下のwarningデザインや記号のレイヤーなど）を正確に表現しつつ、クイズ画面へのスムーズな遷移と、バックエンドへ渡すためのモード選択機能をフロントエンド側に構築する。

### 実施内容
- [ ] 上下配置の「warningデザイン」のUI実装
- [ ] タイトルセクション（英語・日本語）および背面の「記号デザイン」のUI実装（レイヤー順序の制御）
- [ ] キャッチフレーズの配置
- [ ] モード選択コンポーネントの作成（選択形式 / 入力形式の切り替えロジックの準備）
- [ ] ゲーム開始ボタンの作成とクイズ画面への遷移処理（Vite/Reactベース）
- [ ] 画面全体のレスポンシブ・アクセシビリティ対応および0.1秒単位のインタラクション調整

### 完了条件
- [ ] デザイン通りに上下のwarning、記号、タイトルが正しいレイヤー順序（Z軸）で重なって表示されること
- [ ] モード選択によって状態（State）が正しく保持され、ゲーム開始時に次の画面へ渡せる準備ができていること
- [ ] ゲーム開始ボタンを押した際に、クイズ画面（既存またはモック画面）へスムーズに遷移すること
- [ ] コンソールにエラーや、不自然なレイアウト崩れがないこと

### 影響範囲
- フロントエンドのメインルーター・画面遷移ロジック
- スタートページ用コンポーネント（新規作成）
- グローバルStateまたはクイズ画面への引数（選択されたモードの受け渡し）
