Issueタイトル: `[Backend](4): 問題自動生成ロジックの実装`

### 背景
モック API だけではゲーム性が成立しないため、正解文字列とノイズを組み合わせた問題を自動生成する仕組みが必要になる。毎回異なる問題が返ることで、プレイ体験の再現性とリプレイ性を高められる。

### 実施内容
- [ ] ランダムな正解文字列の生成ロジック作成
- [ ] 似て非なるダミー文字列の生成ロジック作成
- [ ] ランダムな記号や文字列ノイズの生成ロジック作成
- [ ] 生成した要素を `noise_text` として結合する処理の作成
- [ ] API を叩くたびに異なるデータが返ることの確認

### 完了条件
- [ ] API 実行ごとに異なる問題データが返ること
- [ ] `hint` と `correct_string` の関係がゲーム仕様に合っていること
- [ ] `noise_text` に正解文字列とノイズが含まれていること

### 影響範囲
- バックエンドの問題生成処理
- API のレスポンス内容
- フロントエンドでのハイライト対象文字列
